Email templates

Transactional email pattern

Email clients (especially Outlook) don't render modern CSS. The component vocabulary from the web prototype does not survive the send. This page is the email-safe subset of the Invenio system — table layout, inline styles, web-safe colors, bulletproof CTAs — and a reusable source template.

01 — Rules

What email clients actually support

The rules below are in order of "most likely to bite you." Outlook is the worst offender — desktop Outlook uses Word's rendering engine, which is stuck in 2007.

Do
Use <table> for layout.
Flexbox and grid do not render in Outlook. Tables are the only reliable layout primitive. All columns, rows, padding — tables.
Do
Inline every style.
Gmail and some corporate clients strip <style> blocks. Put every CSS property on the element itself via style="...".
Do
Use bulletproof CTA buttons.
A link styled as a button with padding + background won't render in Outlook. Wrap a nested table inside an anchor — the table gives the padding even when CSS fails.
Do
Cap width at 600px.
The historical safe width for desktop clients. Wider emails get clipped. Use max-width: 600px + width="100%" on the outer table.
Don't
Use gradients.
The brand mark's sky → cyan gradient does not render in Outlook. Ship the mark as a raster PNG for email; flatten to solid #0369A1.
Don't
Rely on SVG images.
Gmail strips SVG. Export the mark as a 2x PNG (72×72 rendered, 144×144 exported) and reference via absolute URL.
Don't
Use custom web fonts alone.
Outlook and Yahoo Mail ignore @font-face. Specify Inter + a system-sans fallback: font-family: 'Inter', Helvetica, Arial, sans-serif.
Don't
Use background-image on divs.
VML is required for Outlook background images. Prefer solid fills or accept that the background disappears in Outlook.

03 — Safe palette

Email-safe colors

Flatten the brand gradient to a single solid color for email. Use these hex values directly — do not use CSS custom properties (var()) because they don't resolve inline.

Token Hex Use
brand / primary#0369A1CTA button, mark background, link color
text / primary#1E293BHeading, body copy
text / muted#64748BSecondary text, descriptions
text / subtle#94A3B8Footer, metadata
surface#FFFFFFEmail body background
raised#F8FAFCHeader bar, footer strip, detail tables
border#E2E8F0Dividers between sections
